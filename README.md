# AlphaGenome Prototype – Local Test App

Минимальный прототип для экспериментов с **AlphaGenome** в отдельном проекте.
Эта папка не интегрирована в `seqanalysis.org` и предполагается как отдельный
локальный сервис, который ты потом перенесёшь в `c:\dev\...` и будешь развивать там.

## Что здесь есть

- `app.py` — простой веб‑сервер (Flask) с одной страницей:
  - форма для ввода DNA‑последовательности;
  - по нажатию кнопки запрос отправляется на сервер;
  - сервер вызывает AlphaGenome через Python‑клиент и считает очень простой
    summary по предсказаниям (для DNASE в одном UBERON‑тканевом терме);
  - на странице показывается длина последовательности и усреднённые значения
    по трекам.
- `requirements.txt` — зависимости (`alphagenome`, `flask` и т.п.).
- `templates/index.html` — минималистичный интерфейс.

**Важно:**

- API‑ключ **не** захардкожен в коде. Он берётся из переменной окружения
  `ALPHA_GENOME_API_KEY`. Сам ключ лучше хранить отдельно и не класть в Git.
- Этот прототип предназначен **для локальных экспериментов**. Для любой
  интеграции в публичный сайт или платные услуги нужно дополнительно проверить
  лицензии и ограничения AlphaGenome.

## Быстрый старт (локально)

1. Создай и активируй виртуальное окружение (опционально, но желательно):

   ```bash
   python -m venv .venv
   # PowerShell:
   .venv\Scripts\Activate.ps1
   # cmd:
   # .venv\Scripts\activate.bat
   ```

2. Установи зависимости:

   ```bash
   pip install -r requirements.txt
   ```

3. Установи переменную окружения с API‑ключом AlphaGenome:

   ```bash
   # PowerShell
   $env:ALPHA_GENOME_API_KEY = "YOUR_ALPHA_GENOME_KEY_HERE"
   ```

4. Запусти сервер:

   ```bash
   python app.py
   ```

5. Открой в браузере:

   ```text
   http://127.0.0.1:5000/
   ```

Вставь короткую DNA‑последовательность (например, `GATTACA`), нажми **Run AlphaGenome** —
и увидишь простой summary по DNASE‑предсказаниям.

## Как это работает внутри (очень кратко)

- При старте `app.py`:
  - читает `ALPHA_GENOME_API_KEY` из окружения;
  - создаёт `dna_model = dna_client.create(api_key)`.
- При отправке формы:
  - берёт последовательность, чистит/uppercase,
  - центрирует и дополняет `N` до `dna_client.SEQUENCE_LENGTH_1MB`,
  - вызывает `dna_model.predict_sequence(..., requested_outputs=[OutputType.DNASE], ontology_terms=['UBERON:0002048'])`,
  - усредняет `output.dnase.values` по длине последовательности и выводит средние значения по каждому треку.

Если захочешь развивать прототип:

- добавить выбор `OutputType` и `ontology_terms` из выпадающего списка;
- сохранять результаты в локальный `.csv` или базу;
- добавить авторизацию, если будет удалённый доступ.
